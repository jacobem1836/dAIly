# Plan 08-02: Capture Sender in Signal Metadata

**Goal**: Update `_capture_signal()` and the `expand` callsite in the voice graph so sender email is stored in `signal_log.metadata_json`, enabling per-sender aggregation.
**Requires**: Plan 08-01 (defines the sender key format the ranker expects)
**Touches**:
- `src/daily/orchestrator/nodes.py` (modify `_capture_signal` and `summarise_thread_node`)
- `tests/test_orchestrator_nodes.py` or equivalent existing test file (update/add tests)

## Context

Per RESEARCH.md "Critical Finding": `signal_log.metadata_json` is always NULL today because `_capture_signal()` never passes metadata. Without sender in metadata, the adaptive ranker from Plan 08-01 has nothing to aggregate.

This plan wires sender into the existing `expand` capture (the only active email-targeted signal). `respond_node`'s `follow_up` signal remains senderless (it is not email-specific). `skip` and `re_request` capture points are explicitly deferred (per resolved Open Question 2).

## Changes Required

### `src/daily/orchestrator/nodes.py`

**1. Update `_capture_signal` signature and body:**

Add optional `sender: str | None = None` parameter. When provided, normalise via `sender.lower().strip()` and pass `metadata={"sender": normalised}` to `append_signal()`. When `None`, do not pass metadata (preserves existing behaviour for `follow_up`).

```python
async def _capture_signal(
    user_id: int,
    signal_type: SignalType,
    target_id: str | None = None,
    sender: str | None = None,
) -> None:
    # ... existing session setup ...
    metadata = None
    if sender:
        metadata = {"sender": sender.lower().strip()}
    await append_signal(
        user_id=user_id,
        signal_type=signal_type,
        session=session,
        target_id=target_id,
        metadata=metadata,
    )
```

Preserve the existing try/except/logging and session management exactly as-is.

**2. Update the `summarise_thread_node` callsite:**

Before `asyncio.create_task(_capture_signal(...))` for the `expand` signal, look up the sender from `state.email_context` by `message_id`:

```python
sender = next(
    (e["sender"] for e in state.email_context if e.get("message_id") == message_id),
    None,
)
asyncio.create_task(
    _capture_signal(
        state.active_user_id,
        SignalType.expand,
        target_id=message_id,
        sender=sender,
    )
)
```

If `state.email_context` may contain entries without `message_id` or `sender` keys, use `e.get(...)` defensively and fall back to `None`.

**3. `respond_node`:** No change — `follow_up` callsite does not pass `sender` (it is not email-specific).

**4. Skip / re_request:** No new capture points added. Deferred.
