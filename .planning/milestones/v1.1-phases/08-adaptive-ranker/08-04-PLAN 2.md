# Plan 08-04: Wire db_session Through Pipeline and Scheduler

**Goal**: Add optional `db_session` parameter to `run_briefing_pipeline()` and open a session inside the scheduler's `_build_pipeline_kwargs()` so the cron pipeline activates adaptive ranking end-to-end.
**Requires**: Plans 08-01, 08-02, 08-03
**Touches**:
- `src/daily/briefing/pipeline.py`
- `src/daily/briefing/scheduler.py`
- `tests/test_briefing_pipeline.py` (extend)
- `tests/test_briefing_scheduler.py` (extend)

## Context

Plans 08-01 through 08-03 built the ranker, signal capture, and wiring into `build_context`. This plan closes the last gap: `run_briefing_pipeline()` receives a `db_session` and passes it through to `build_context()`, and the scheduler opens a dedicated session per cron run.

Per RESEARCH.md Pattern 5, the session is opened inside `_build_pipeline_kwargs` using the existing `async_session()` factory pattern already used elsewhere in that function.

### `src/daily/briefing/scheduler.py`

In `_scheduled_pipeline_run`, open a dedicated session and pass it through:

```python
async def _scheduled_pipeline_run(user_id: int) -> None:
    settings = Settings()
    kwargs: dict = {}
    try:
        kwargs = await _build_pipeline_kwargs(user_id, settings)
        async with async_session() as session:
            await run_briefing_pipeline(
                user_id=user_id,
                db_session=session,
                **kwargs,
            )
    except Exception:
        logger.exception("Scheduled briefing pipeline failed for user %d", user_id)
    finally:
        redis = kwargs.get("redis")
        if redis is not None:
            await redis.aclose()
```

Rationale: session lifetime is scoped to the pipeline run; opened and closed cleanly via `async with`. This matches the existing `async with async_session() as session:` pattern elsewhere in this module.

Do NOT modify `_build_pipeline_kwargs` to include the session in its returned dict — the session is passed separately by `_scheduled_pipeline_run` so its lifetime is visibly scoped at the call site.

**On-demand path (`get_or_generate_briefing`):** No change in this plan. The on-demand path does not currently pass a session and will simply run in pure-heuristic mode (backward-compatible). A follow-up phase can wire it if desired.
