<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Voice loop -> detached asyncio task | The extraction task runs after the voice loop's main body exits; no exception from the task can propagate to the caller, but must not leak DB connections. |
| session_history (in-memory) -> extraction LLM | Accumulated user+assistant turns are raw conversation text; they flow into the extraction prompt and must be treated as untrusted text (already mitigated by Plan 02's strict JSON parsing). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-14 | Denial of Service | Extraction task blocks voice shutdown | mitigate | `asyncio.create_task(_run_memory_extraction())` is fire-and-forget — never awaited. The finally block completes immediately after scheduling. Task 1 acceptance criteria includes a grep guard that `await ` does not precede the `create_task` call. |
| T-09-15 | Information Disclosure | session_history logged in full on failure | mitigate | The wrapper `_run_memory_extraction` logs only the exception class/message (`"memory extraction task failed for user=%d: %s"`). The full transcript is never passed to `logger.*` calls. Inner extraction (Plan 02) also truncates fact_text in debug logs. |
| T-09-16 | Tampering | Injected memory fed back into extraction (hallucination loop) | mitigate | `session_history` only contains per-turn `user_input` and LLM `response_content`, NEVER the injected `state.user_memories`. Extraction prompt explicitly excludes assistant turns from fact extraction ("Only extract facts stated by the user"). Dedup (Plan 02) is the second guard. `test_no_hallucination_loop` asserts this contract. |
| T-09-17 | Repudiation | Extraction task fails silently leaving no record | accept | Per CONTEXT.md specifics #2, extraction MUST never raise. `logger.warning` on failure is the audit trail. Storage of failed-extraction events is out of scope. |
| T-09-18 | Resource leak | `_async_session()` connection not released on task cancellation | mitigate | `async with _async_session() as mem_session:` guarantees release on success, exception, AND task cancellation — the `async with` protocol triggers `__aexit__` on CancelledError. No additional handling required. |
</threat_model>
