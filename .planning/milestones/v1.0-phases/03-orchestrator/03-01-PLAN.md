1. Create `src/daily/profile/__init__.py` (empty).

2. Create `src/daily/profile/models.py`:
   - Import `Base` from `daily.db.models`
   - `UserProfile(Base)` ORM model:
     - `__tablename__ = "user_profile"`
     - `id: Mapped[int] = mapped_column(primary_key=True)`
     - `user_id: Mapped[int] = mapped_column(ForeignKey("users.id"), unique=True)`
     - `preferences: Mapped[dict] = mapped_column(JSONB, default=dict, server_default="{}")` (import JSONB from `sqlalchemy.dialects.postgresql`)
     - `created_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())`
     - `updated_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now(), onupdate=func.now())`
   - `UserPreferences(BaseModel)` Pydantic model (NOT ORM — this is the typed view of the JSONB):
     - `tone: Literal["formal", "casual", "conversational"] = "conversational"`
     - `briefing_length: Literal["concise", "standard", "detailed"] = "standard"`
     - `category_order: list[str] = ["emails", "calendar", "slack"]`
   Per D-04: JSONB allows schema evolution without migrations. UserPreferences is validated on read, not enforced by DB schema.

3. Create `src/daily/profile/service.py`:
   - `async def load_profile(user_id: int, session: AsyncSession) -> UserPreferences`:
     - SELECT from user_profile WHERE user_id = user_id
     - If no row, return `UserPreferences()` (defaults)
     - If row exists, return `UserPreferences.model_validate(row.preferences)`
   - `async def upsert_preference(user_id: int, key: str, value: str, session: AsyncSession) -> UserPreferences`:
     - SELECT existing UserProfile row
     - If no row, create one with default preferences dict
     - Update the specific key in the preferences dict: `profile.preferences[key] = value`
     - For "category_order", parse value as comma-separated list: `value.split(",")`
     - `await session.commit()`
     - Return `UserPreferences.model_validate(profile.preferences)`

4. Create `tests/test_profile_service.py` with tests per behavior block above. Use an in-memory SQLite or test Postgres fixture consistent with existing test patterns. If existing tests use a real Postgres test DB, follow that pattern. If they mock the session, follow that.
