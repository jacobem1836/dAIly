    2. **src/daily/vault/crypto.py**: Implement exactly as specified in RESEARCH.md Pattern 1:
    ```python
    from cryptography.hazmat.primitives.ciphers.aead import AESGCM
    import os, base64

    def encrypt_token(plaintext: str, key: bytes) -> str:
        if len(key) != 32:
            raise ValueError("Key must be 32 bytes for AES-256")
        aesgcm = AESGCM(key)
        nonce = os.urandom(12)  # 96-bit nonce, fresh every call
        ciphertext = aesgcm.encrypt(nonce, plaintext.encode(), None)
        return base64.b64encode(nonce + ciphertext).decode()

    def decrypt_token(encrypted: str, key: bytes) -> str:
        if len(key) != 32:
            raise ValueError("Key must be 32 bytes for AES-256")
        data = base64.b64decode(encrypted.encode())
        nonce, ciphertext = data[:12], data[12:]
        aesgcm = AESGCM(key)
        return aesgcm.decrypt(nonce, ciphertext, None).decode()
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env -> application | Vault key loaded from environment; if compromised, all tokens decryptable |
| OAuth provider -> callback server | Auth code received over localhost; could be intercepted on shared machine |
| Application -> PostgreSQL | Encrypted tokens cross this boundary; DB compromise yields only ciphertext |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation |
|-----------|----------|-----------|----------|-------------|------------|
| T-1-01 | Information Disclosure | vault/crypto.py | HIGH | mitigate | AES-256-GCM encryption before any DB write; key validated at startup (Task 2) |
| T-1-02 | Tampering | vault/crypto.py | HIGH | mitigate | GCM authentication tag detects tampering; InvalidTag raised on corrupted ciphertext (Task 2) |
| T-1-03 | Information Disclosure | .env / VAULT_KEY | MED | accept | M1 uses .env for secrets; production secrets management is M2+. .env is gitignored. |
| T-1-04 | Tampering | db/models.py | MED | mitigate | No raw_body column exists in schema — SEC-04 enforced architecturally; tested in test_schema.py (Task 2) |
| T-1-05 | Information Disclosure | logs | MED | mitigate | SQLAlchemy echo=False; no token values in log output; encrypted values only in DB |
