    ```python
    async def run_voice_session(user_id: int = 1) -> None:
        """Run an interactive voice session with the orchestrator.

        Mirrors _run_chat_session() from cli.py but replaces input()/print()
        with VoiceTurnManager (mic capture + TTS playback).

        Per D-01: daily voice CLI command mirrors daily chat structure.
        Per D-02: Same orchestrator graph (build_graph, run_session) — voice is I/O only.
        Per D-11: AsyncPostgresSaver replaces MemorySaver for persistent state.
        """
    ```

    Implementation (following _run_chat_session pattern exactly):

    1. **Setup phase:**
       - Load Settings.
       - Validate `deepgram_api_key` and `cartesia_api_key` are set (print error and return if not).
       - Resolve email adapters via `_resolve_email_adapters(user_id, settings)` (import from cli.py).
       - Call `set_email_adapters(adapters)`.

    2. **AsyncPostgresSaver (D-11):**
       - Use `async with AsyncPostgresSaver.from_conn_string(settings.database_url_psycopg) as checkpointer:` (Pitfall 4: use psycopg URL, NOT asyncpg URL).
       - Call `await checkpointer.setup()` (idempotent — safe every session start).
       - Build graph: `graph = build_graph(checkpointer=checkpointer)`.

    3. **Session init:**
       - `config = await create_session_config(user_id)`.
       - Create Redis connection, load initial state via `initialize_session_state()`, close Redis.

    4. **Voice I/O setup:**
       - Create `TTSPipeline(api_key=settings.cartesia_api_key)`.
       - Create `STTPipeline(api_key=settings.deepgram_api_key)`.
       - Create `VoiceTurnManager(tts=tts_pipeline, stt=stt_pipeline)`.
       - Create `listen_stop = asyncio.Event()` (global stop for STT listener).
       - Start STT: `await turn_manager.start_stt(listen_stop)`.

    5. **First turn — speak briefing from cache (VOICE-03 sub-1s):**
       - If `initial_state["briefing_narrative"]` is non-empty, speak it: `await turn_manager.speak(narrative)`.
       - Print "dAIly: [briefing spoken]" to terminal for log.

    6. **Main voice loop:**
       ```python
       first_turn = True
       while True:
           # Wait for user utterance from Deepgram
           user_input = await turn_manager.wait_for_utterance()
           if user_input.lower().strip() in ("exit", "quit"):
               print("Session ended.")
               break

           # Run through orchestrator (same as chat)
           result = await run_session(
               graph, user_input, config,
               initial_state=initial_state if first_turn else None,
           )
           first_turn = False

           # Check for approval interrupt
           graph_state = await graph.aget_state(config)
           if graph_state.next:
               # Voice approval flow: speak draft, listen for confirm/reject/edit
               await _handle_voice_approval(turn_manager, graph, graph_state, config)
           else:
               # Normal response: speak it
               messages = result.get("messages", [])
               if messages:
                   content = messages[-1].content if hasattr(messages[-1], "content") else str(messages[-1])
                   await turn_manager.speak(content)
       ```

    7. **Voice approval flow** (`_handle_voice_approval`):
       - Extract preview from interrupt payload (same logic as `_handle_approval_flow` in cli.py).
       - Speak the draft preview via `turn_manager.speak(preview_text)`.
       - Speak prompt: "Confirm, reject, or describe changes."
       - `decision_input = await turn_manager.wait_for_utterance()`.
       - Parse decision via `_parse_approval_decision(decision_input)` (reuse from cli.py).
       - Resume graph: `await graph.ainvoke(Command(resume=decision), config=config)`.
       - Handle edit loop (same pattern as cli.py while loop).

    8. **Cleanup:**
       - Wrap main loop in try/finally.
       - `listen_stop.set()` to stop STT listener.
       - `await turn_manager.stop()`.
       - Print "Voice session ended."

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Voice transcript -> orchestrator graph | User speech (via Deepgram) becomes LLM input |
| AsyncPostgresSaver -> PostgreSQL | Graph state persisted to database |
| Voice approval -> action executor | Spoken "confirm" triggers real external actions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Elevation of Privilege | Voice approval bypass | mitigate | Approval flow reuses same `_parse_approval_decision` logic as CLI; graph interrupt pattern cannot be bypassed (approval_node uses interrupt()); no code path from voice input directly to execute_node |
| T-05-11 | Information Disclosure | Session state in PostgreSQL | mitigate | AsyncPostgresSaver uses same database as existing app; thread_id scoped per user per day (T-03-04); no new access surface |
| T-05-12 | Tampering | Transcript injection via adversarial speech | mitigate | Transcript passes through same route_intent keyword filter and OrchestratorIntent JSON validation as chat input (SEC-05); no new unvalidated input path |
| T-05-13 | Denial of Service | WebSocket connection exhaustion | accept | Single-user M1; one Deepgram + one Cartesia connection per session; well within service limits |
</threat_model>
