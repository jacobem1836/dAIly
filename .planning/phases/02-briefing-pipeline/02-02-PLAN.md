1. Constants (Claude's discretion on exact values, research suggests):
   - `WEIGHT_VIP = 40` (VIP override — always maximum sender weight)
   - `WEIGHT_DIRECT = 10` (user in To: field)
   - `WEIGHT_CC = 2` (user in CC/BCC)
   - `WEIGHT_KEYWORD_HIT = 8` (per keyword match in subject)
   - `WEIGHT_RECENCY_MAX = 15` (linear decay over 24h)
   - `WEIGHT_THREAD_ACTIVE = 5` (thread_id appears 3+ times in batch)
   - `DEADLINE_KEYWORDS = frozenset(["urgent", "action required", "deadline", "by eod", "due today", "asap", "time sensitive", "response needed"])`

2. `score_email(email: EmailMetadata, vip_senders: frozenset[str], user_email: str, now: datetime, thread_counts: dict[str, int]) -> float`:
   - sender_weight: WEIGHT_VIP if `email.sender` in vip_senders, else WEIGHT_DIRECT if `user_email.lower()` in `email.recipient.lower()`, else WEIGHT_CC
   - keyword_weight: sum of WEIGHT_KEYWORD_HIT for each keyword in DEADLINE_KEYWORDS found in `email.subject.lower()`
   - recency_weight: `WEIGHT_RECENCY_MAX * max(0, (24 - hours_old) / 24)` where `hours_old = (now - email.timestamp).total_seconds() / 3600`. Ensure both datetimes are UTC-aware.
   - thread_activity_weight: WEIGHT_THREAD_ACTIVE if `thread_counts.get(email.thread_id, 0) >= 3` else 0
   - return sum of all four weights

3. `rank_emails(emails: list[EmailMetadata], vip_senders: frozenset[str], user_email: str, top_n: int = 5) -> list[RankedEmail]`:
   - Compute `thread_counts`: dict mapping thread_id to count of emails with that thread_id
   - Score all emails using `score_email`
   - Sort by score descending
   - Return top `top_n` as `RankedEmail(metadata=email, score=score)` objects
   - Per D-02/D-05: this function handles the "rank all, return top-N" logic
