# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Briefing Pipeline.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Note

Phase 2 has **no web frontend surface**. The project is Python backend-only; the
ROADMAP explicitly defers all web dashboard work to M2 (DASH-01 through DASH-03 are
v2 requirements). The "UI" for this phase is the **CLI terminal interaction surface**
and the **briefing narrative text contract** (the spoken-English output the narrator
LLM must produce).

Design system tooling (shadcn, Tailwind, component libraries) is **not applicable**.

## Spacing Scale

Not applicable to terminal output. CLI output uses line breaks and indentation only.

| Convention | Value | Usage |
|------------|-------|-------|
| Indent level | 2 spaces | Sub-items in multi-line CLI output |
| Section break | 1 blank line | Between logical output groups |
| Major break | 2 blank lines | Between top-level CLI command outputs |

Exceptions: none

---

## Typography

Terminal output only — no font size or weight control. Conventions govern structure instead.

| Role | Convention | Usage |
|------|------------|-------|
| Heading | ALL CAPS prefix, e.g. `BRIEFING:` | Section labels in terminal output |
| Body | Plain lowercase prose | Narrative content, status messages |
| Label | `key: value` format | Config listing, VIP list output |
| Error | `Error: ` prefix, plain prose | Error messages (see Copywriting) |

Line length target: 80 characters. Wrap at word boundaries for multi-line output.

---

## Color

Terminal only — no hex color control. ANSI color usage:

| Role | ANSI | Usage |
|------|------|-------|
| Success | green (32) | Confirmation messages: "Briefing cached.", "VIP added." |
| Error | red (31) | Error messages: any `Error:` prefixed line |
| Warning | yellow (33) | Warning states: cache miss fallback, schedule update notice |
| Neutral | default | All other output — config listings, narrative preview |

Accent reserved for: success confirmations and error lines only. Never applied to
narrative content or config listings.

Note: ANSI output must respect `NO_COLOR` environment variable and non-TTY pipes
(strip color codes when stdout is not a TTY).

## CLI Command Surface

All commands follow the pattern `daily <noun> <verb> [args]`. No positional-only
commands except where established by Phase 1 (e.g. `daily oauth connect`).

### Commands introduced in Phase 2

| Command | What it does |
|---------|-------------|
| `daily config set briefing.schedule_time HH:MM` | Update precompute schedule time |
| `daily config set briefing.email_top_n N` | Set how many top-ranked emails fetch bodies |
| `daily config get briefing` | Print current briefing config values |
| `daily vip add <email>` | Add sender to VIP list |
| `daily vip remove <email>` | Remove sender from VIP list |
| `daily vip list` | Print current VIP senders |
| `daily briefing run` | Trigger pipeline manually (dev/debug use) |
| `daily briefing show` | Print cached briefing narrative to terminal |

### Output format for `daily briefing show`

```
BRIEFING: <date>
Generated: <ISO-8601 timestamp>  |  Cached

<narrative paragraph 1 — critical emails>

<narrative paragraph 2 — calendar>

<narrative paragraph 3 — Slack>
```

If no cached briefing exists (cache miss):

```
No cached briefing for today. Generating now — this may take 10–20 seconds...

BRIEFING: <date>
Generated: <ISO-8601 timestamp>  |  Live

<narrative>
```

## Briefing Narrative Text Contract

The narrator LLM output must satisfy these constraints (source: CONTEXT.md D-06, D-07, D-08).

| Property | Contract |
|----------|----------|
| Format | Flowing prose — no bullet points, no numbered lists, no markdown |
| Structure | Three paragraphs in order: (1) critical emails, (2) calendar, (3) Slack |
| Length | 225–300 words total (~90–120 seconds at average TTS pace) |
| Register | Spoken English — contractions allowed, direct address ("You have...") |
| Empty source | One sentence: "Nothing notable in [source] today." |

The narrative text is what Phase 5 (Voice) will pass to Cartesia TTS. It must be
clean plain text with no markdown, no special characters, no parenthetical
formatting annotations.

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Primary CTA | `daily briefing show` | CONTEXT.md D-16 |
| Empty briefing heading | `No cached briefing for today.` | CONTEXT.md D-15 |
| Empty briefing body | `Generating now — this may take 10–20 seconds...` | CONTEXT.md D-15 |
| Cache miss label | `Live` (vs `Cached` on hit) | Inferred from D-14 |
| Empty source (in narrative) | `Nothing notable in [source] today.` | CONTEXT.md D-08 |
| Schedule updated | `Briefing schedule updated. Next run: [time].` | Default |
| VIP added | `[email] added to VIP senders.` | Default |
| VIP removed | `[email] removed from VIP senders.` | Default |
| VIP list empty | `No VIP senders configured. Use: daily vip add <email>` | Default |
| Pipeline success | `Briefing cached for [date]. Narrative: [N] words.` | Default |
| Error: no connection | `Error: No connected accounts found. Run daily oauth connect first.` | Default |
| Error: LLM failure | `Error: Narrative generation failed. Check OPENAI_API_KEY and retry.` | Default |
| Error: Redis unavailable | `Error: Cache unavailable. Check Redis connection and retry.` | Default |
| Error: invalid time format | `Error: Schedule time must be in HH:MM format (e.g. 05:00).` | Default |
| Error: invalid top-N | `Error: email_top_n must be a positive integer.` | Default |

### Destructive actions in Phase 2

| Action | Confirmation approach |
|--------|-----------------------|
| `daily vip remove <email>` | No confirmation prompt — non-destructive data (VIP list is additive; removal is low-risk and reversible with `daily vip add`). |

No irreversible destructive actions in Phase 2. No confirmation prompts required.
