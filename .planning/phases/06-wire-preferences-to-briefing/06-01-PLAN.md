<objective>
Wire user preferences (tone, briefing_length, category_order) into the scheduled briefing pipeline so the precomputed morning briefing respects user-configured preferences -- not just the CLI chat session.

Purpose: Closes the PERS-01 gap. Phase 3 built UserProfile ORM + load_profile() and Phase 3 also made narrator.py accept an optional preferences parameter. But the pipeline.py and scheduler.py never call load_profile() or pass preferences through. This plan completes that wiring.

Output: Modified pipeline.py, scheduler.py, narrator.py (import fix), and updated tests proving preferences flow end-to-end through the cron-triggered pipeline.
</objective>
