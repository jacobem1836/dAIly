# Plan 08-01: Adaptive Ranker Core Module

**Goal**: Create `adaptive_ranker.py` exporting `get_sender_multipliers()` with signal aggregation, exponential decay, sigmoid normalisation, cold-start, and graceful degradation.
**Requires**: None (foundation plan)
**Touches**:
- `src/daily/profile/adaptive_ranker.py` (new)
- `tests/test_adaptive_ranker.py` (new)

## Context

Phase 8 replaces static heuristic ranking with a signal-learned per-sender multiplier. This plan delivers the core computation module — queries `signal_log`, applies 14-day exponential decay per signal, sums weighted contributions per sender, and sigmoid-normalises to a 0.0–2.0 range where raw score 0 maps exactly to multiplier 1.0 (neutral).

The module must never raise: any DB error returns `{}` so the briefing pipeline always delivers (BRIEF-01 contract).

## Changes Required

### `src/daily/profile/adaptive_ranker.py` (NEW)

Create module with:

**Module constants:**
```python
HALF_LIFE_DAYS = 14.0
MIN_MULT = 0.0
MAX_MULT = 2.0
DEFAULT_MIN_SIGNALS = 30

SIGNAL_WEIGHTS: dict[str, float] = {
    "re_request": 1.0,
    "expand": 0.5,
    "follow_up": 0.3,
    "correction": -0.3,
    "skip": -0.5,
}
```

**Private helpers:**
- `_decay_weight(created_at: datetime) -> float` — returns `exp(-ln(2) * days_old / 14)`. Defensively coerces naive `created_at` to UTC (see Pitfall 2 in RESEARCH).
- `_sigmoid_neutral_at_one(raw_score: float) -> float` — returns `2.0 * (1 / (1 + exp(-raw_score)))`. Raw score 0 → 1.0 exactly (per resolved Open Question 1). Range (0.0, 2.0).
- `_compute_multipliers(rows: list[tuple[str, str, datetime]]) -> dict[str, float]` — iterate rows (sender, signal_type, created_at), accumulate `decay_weight * SIGNAL_WEIGHTS[signal_type]` per sender, then sigmoid-normalise each accumulated raw score. Skip signal_types not in SIGNAL_WEIGHTS.

**Public function:**
```python
async def get_sender_multipliers(
    user_id: int,
    session: AsyncSession,
    min_signals: int = DEFAULT_MIN_SIGNALS,
) -> dict[str, float]:
```
Steps:
1. Wrap entire body in `try/except Exception` — on any exception, log warning via `logger.warning(...)` and return `{}`.
2. Count total signals for user: `select(func.count()).select_from(SignalLog).where(SignalLog.user_id == user_id)`. If `< min_signals`, return `{}`.
3. Query senders with NULL guard:
   ```python
   stmt = (
       select(
           cast(SignalLog.metadata_json["sender"].astext, String).label("sender"),
           SignalLog.signal_type,
           SignalLog.created_at,
       )
       .where(SignalLog.user_id == user_id)
       .where(SignalLog.metadata_json.isnot(None))
       .where(SignalLog.metadata_json["sender"].astext.isnot(None))
   )
   ```
   (Use `.astext` for the JSONB text accessor — the existing SQLAlchemy idiom for JSONB string extraction.)
4. Fetch rows, call `_compute_multipliers`, return result.
5. Keys in the returned dict must be normalised: `sender.lower().strip()` (matches ranker's sender lookup — Pitfall 5).

**Module docstring:** Reference INTEL-01, decisions from 08-CONTEXT.md, and the "never raises" contract.

### `tests/test_adaptive_ranker.py` (NEW)

Pytest-asyncio tests using `AsyncMock(spec=AsyncSession)`. Required test cases:

1. `test_cold_start_returns_empty` — count query returns 5, expect `{}`.
2. `test_db_error_returns_empty` — `session.execute` raises `OperationalError`, expect `{}` and a warning logged.
3. `test_null_metadata_excluded` — query returns only rows where metadata has sender; assert no error on empty result, returns `{}`.
4. `test_sigmoid_zero_score` — directly test `_sigmoid_neutral_at_one(0.0) == pytest.approx(1.0)`.
5. `test_decay_half_life` — directly test `_decay_weight(now - 14 days) == pytest.approx(0.5, abs=0.01)`.
6. `test_compute_multipliers_engaged_sender` — feed fake rows: alice with 5 expand signals all recent (decay≈1.0) → raw≈2.5 → multiplier>1.8. bob with 5 skip signals recent → raw≈-2.5 → multiplier<0.25.
7. `test_unknown_signal_type_ignored` — rows include signal_type not in SIGNAL_WEIGHTS, should not contribute (no KeyError).
8. `test_sender_key_normalisation` — sender `"  ALICE@example.com  "` in metadata, result dict key is `"alice@example.com"`.

Mock pattern: count query returns int; detail query returns iterable of tuple-like rows. Use two `side_effect` entries on `session.execute`.
