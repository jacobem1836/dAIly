# Plan 01: Structured Logging Infrastructure

<objective>
All application logging emits structured JSON with consistent fields, and log verbosity is configurable via the LOG_LEVEL environment variable — zero call-site changes to existing modules.

Purpose: The codebase has 17+ modules using logging.getLogger(__name__). A JSONFormatter at the handler level makes all existing logger calls emit structured JSON transparently. A ContextAdapter factory enables modules to inject user_id and stage into log context.
Output: src/daily/logging_config.py, updated config.py and main.py, test suite.
</objective>

    1. Add `log_level: str = "INFO"` field to `Settings` in `src/daily/config.py` (per D-04). Pydantic-settings auto-maps `LOG_LEVEL` env var.

    2. Create `src/daily/logging_config.py` with:
       - `JSONFormatter(logging.Formatter)`: override `format()` to return `json.dumps(...)` with fields per D-02: `ts` (ISO-8601 UTC from `record.created`), `level` (record.levelname), `module` (record.name), `msg` (record.getMessage()), `ctx` (from `getattr(record, "ctx", {})`). If `record.exc_info`, include `exc` field via `self.formatException()`. Use `datetime.fromtimestamp(record.created, tz=timezone.utc).isoformat()` for ts.
       - `ContextAdapter(logging.LoggerAdapter)`: override `process()` to inject `ctx` dict from `self.extra["ctx"]` into `kwargs["extra"]["ctx"]`.
       - `configure_logging(log_level: str = "INFO") -> None`: clear root handlers (`root.handlers.clear()`), create `StreamHandler` with `JSONFormatter`, add to root logger, set level via `getattr(logging, log_level.upper(), logging.INFO)`.
       - `make_logger(name: str, **ctx_fields) -> logging.LoggerAdapter`: return `ContextAdapter(logging.getLogger(name), extra={"ctx": ctx_fields})`.

    3. Wire `configure_logging()` into `main.py` lifespan:
       - Import `configure_logging` from `daily.logging_config`.
       - Call `configure_logging(settings.log_level)` as the FIRST line after `settings = Settings()` in the lifespan startup.
       - This must come before any logger.info() calls so all startup logs are also JSON.

    IMPORTANT: Do NOT modify any other module's logger calls. The formatter intercepts at the handler level — existing `logger.info(...)` calls work unchanged per D-01.
    IMPORTANT: Use `record.name` for module field (not `record.module`) per Pitfall 4 in RESEARCH.md.
    IMPORTANT: Call `root.handlers.clear()` before adding JSON handler per Pitfall 1 in RESEARCH.md.
  </action>
