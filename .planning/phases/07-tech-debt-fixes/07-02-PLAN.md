# Plan 07-02: Bug B — Slack cursor-based pagination

## Objective

Implement D-03 (cursor-based pagination loop in Slack adapter following `next_cursor` until
exhausted) and D-04 (wire cursor support through `context_builder.py:114-118` matching the
existing email pagination pattern).

Currently `SlackAdapter.list_messages` reads `next_cursor` from each channel response but
never follows it (adapter.py:54-88), and `context_builder._fetch_all_messages` explicitly
documents single-page-only behaviour (context_builder.py:114-118). Channels with >100
messages in the 24h window silently lose data.

Purpose: Correctness fix — the briefing must reflect the full Slack mention/DM volume, not
just the first page.

Output: `MessageAdapter.list_messages` accepts `cursor`; Slack adapter loops per channel
following cursors; `context_builder._fetch_all_messages` drives the outer loop until
`next_cursor is None`, mirroring `_fetch_all_emails`.

   ```python
   all_messages: list[MessageMetadata] = []

   for channel_id in channels:
       is_dm = channel_id.startswith("D")
       oldest_ts = since.timestamp()
       channel_cursor: str | None = cursor  # only relevant for first channel on first call

       while True:
           call_kwargs = {
               "channel": channel_id,
               "oldest": oldest_ts,
               "limit": 100,
           }
           if channel_cursor:
               call_kwargs["cursor"] = channel_cursor

           response = await asyncio.to_thread(
               self._client.conversations_history,
               **call_kwargs,
           )

           messages_data = response.get("messages", [])
           response_metadata = response.get("response_metadata", {}) or {}
           raw_cursor = response_metadata.get("next_cursor", "")
           channel_cursor = raw_cursor if raw_cursor else None

           for msg in messages_data:
               ts = msg.get("ts", "")
               timestamp = datetime.fromtimestamp(float(ts), tz=timezone.utc)
               text = msg.get("text", "")
               is_mention = "<@" in text

               all_messages.append(
                   MessageMetadata(
                       message_id=ts,
                       channel_id=channel_id,
                       sender_id=msg.get("user", ""),
                       timestamp=timestamp,
                       is_mention=is_mention,
                       is_dm=is_dm,
                   )
               )

           if channel_cursor is None:
               break

   return MessagePage(messages=all_messages, next_cursor=None)
   ```

```python
async def _fetch_all_messages(
    adapters: list[MessageAdapter], since: datetime
) -> list[MessageMetadata]:
    """Fetch all message metadata from all adapters, handling pagination.

    Loops through pages until next_cursor is None for each adapter.

    Note: SlackAdapter exhausts its own per-channel pagination internally and
    always returns next_cursor=None, so this loop will execute exactly once per
    Slack adapter. The loop shape mirrors `_fetch_all_emails` so future adapters
    that paginate at the adapter level (rather than per-channel) work transparently.

    Args:
        adapters: List of message adapter instances to query.
        since: Only fetch messages after this datetime.

    Returns:
        Combined list of MessageMetadata from all adapters and all pages.
    """
    all_messages: list[MessageMetadata] = []
    for adapter in adapters:
        cursor: str | None = None
        while True:
            page = await adapter.list_messages(channels=[], since=since, cursor=cursor)
            all_messages.extend(page.messages)
            cursor = page.next_cursor
            if cursor is None:
                break
    return all_messages
```
