       ```python
       async def retrieve_relevant_memories(
           user_id: int,
           query_text: str,
           db_session: "AsyncSession",
           top_k: int = 10,
       ) -> list[str]:
           """Retrieve top-K most relevant facts for a user by cosine similarity.

           Returns [] if memory_enabled=False or on any failure (fail-silent).
           Fact text only — no metadata exposed to LLM (D-06 specifics).
           """
           try:
               preferences = await load_profile(user_id, db_session)
               if not preferences.memory_enabled:
                   return []

               client = _get_openai_client()
               query_embedding = await _embed(query_text, client)

               stmt = (
                   select(MemoryFact.fact_text)
                   .where(MemoryFact.user_id == user_id)
                   .order_by(MemoryFact.embedding.cosine_distance(query_embedding))
                   .limit(top_k)
               )
               rows = (await db_session.execute(stmt)).scalars().all()
               return list(rows)
           except Exception as exc:
               logger.warning(
                   "retrieve_relevant_memories: failed for user=%d: %s",
                   user_id, exc,
               )
               return []
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DB -> LLM context | Retrieved `fact_text` values flow into the narrator and response-node system prompts; they must be treated as data, not instructions (no tool calls from narrator per SEC-05). |
| LangGraph checkpointer -> SessionState | Adding a new field to the Pydantic state risks breaking deserialization of old checkpoint rows in production. Mitigated by `Field(default_factory=list)`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-10 | Tampering | Prompt injection via stored fact | mitigate | Facts are wrapped in a clearly-labelled MEMORY_PREAMBLE and placed BEFORE NARRATOR_SYSTEM_PROMPT. Narrator LLM has no tool calls (SEC-05); response node inherits the same "no tools" constraint. Memory preamble explicitly says "Do not invent memories" to reduce model drift on malicious facts. |
| T-09-11 | Information Disclosure | Cross-user memory retrieval | mitigate | `retrieve_relevant_memories` hard-filters `WHERE user_id = :user_id`. No codepath queries MemoryFact without a user_id filter. |
| T-09-12 | Tampering | SessionState checkpoint schema drift | mitigate | `user_memories: list[str] = Field(default_factory=list)` — old checkpoints missing the field load safely with empty default. Pydantic's `BaseModel` defaults handle forward compatibility. |
| T-09-13 | Denial of Service | Slow pgvector query blocks briefing pipeline | mitigate | HNSW index (Plan 01) keeps cosine queries sub-millisecond at M1 scale; `top_k=10` limit bounds result set. Retrieval is guarded by try/except that returns `[]` on any failure — briefing pipeline always delivers (BRIEF-01). |
</threat_model>
