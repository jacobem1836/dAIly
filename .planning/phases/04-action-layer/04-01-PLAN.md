<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output -> OrchestratorIntent | Untrusted LLM string must validate against Literal whitelist |
| User CLI input -> Command(resume=) | User decision string flows into graph state |
| Draft body -> action_log | PII-containing content must not be stored raw |
| Draft recipient -> external API | Recipient must be validated against known contacts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | OrchestratorIntent.action | mitigate | Extended Literal whitelist rejects unknown action types via Pydantic ValidationError (SEC-05) |
| T-04-02 | Elevation of Privilege | execute_node | mitigate | Graph topology enforces draft -> approval -> execute path; no edge from START to execute; interrupt() must fire before execution |
| T-04-03 | Information Disclosure | action_log.body | mitigate | Store content_summary[:200] + SHA-256 hash only; never raw body (D-09/SEC-04) |
| T-04-04 | Tampering | ActionExecutor.validate | mitigate | check_recipient_whitelist rejects unknown addresses before any API call (ACT-06/D-07) |
| T-04-05 | Spoofing | approval_node state | mitigate | thread_id scoped to user_id + date (established Phase 3 pattern); checkpointed state tied to specific user session |
| T-04-06 | Tampering | approval_decision value | accept | Decision string is "confirm"/"reject"/"edit:*" — invalid values result in cancellation (safe default). Low risk as CLI input is from authenticated user. |
</threat_model>
