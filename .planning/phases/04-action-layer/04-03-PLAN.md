```python
class ActionType(str, Enum):
    draft_email = "draft_email"
    draft_message = "draft_message"
    compose_email = "compose_email"
    schedule_event = "schedule_event"
    reschedule_event = "reschedule_event"

class ActionDraft(BaseModel):
    action_type: ActionType
    recipient: str | None = None
    subject: str | None = None
    body: str
    thread_id: str | None = None
    thread_message_id: str | None = None  # RFC 2822 Message-ID for email threading
    channel_id: str | None = None
    event_id: str | None = None
    event_title: str | None = None
    start_dt: datetime | None = None
    end_dt: datetime | None = None
    attendees: list[str] = []

class ActionResult(BaseModel):
    success: bool
    external_id: str | None = None
    error: str | None = None

class ActionExecutor(ABC):
    async def validate(self, draft: ActionDraft) -> None: ...
    async def execute(self, draft: ActionDraft) -> ActionResult: ...
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ActionDraft -> External API | Validated draft content crosses to Gmail/Slack/Calendar/Outlook APIs |
| Decrypted OAuth token -> API call | Token decrypted in-memory only at execution time |
| Known addresses set -> whitelist check | Contact list used for recipient validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Tampering | GmailExecutor.execute | mitigate | validate() with check_recipient_whitelist before any API call (ACT-06) |
| T-04-12 | Tampering | SlackExecutor.execute | mitigate | Channel validated against known_channels set; thread_ts cast to str to prevent silent failure |
| T-04-13 | Information Disclosure | OAuth token in executor | mitigate | Token decrypted in-memory only at API call time via existing Phase 1 vault pattern; never logged or passed to LLM |
| T-04-14 | Denial of Service | Calendar events.insert | accept | No rate limiting on calendar writes in M1; Google API has its own rate limits. Low risk for single-user system. |
| T-04-15 | Tampering | Calendar events.update vs patch | mitigate | Use events.patch exclusively for reschedules; events.update is never called (Pitfall 5 prevention) |
| T-04-16 | Elevation of Privilege | execute_node direct call | mitigate | execute_node only reachable via approval_node in graph topology (enforced by Plan 01 graph edges); executor.validate() is the second gate |
</threat_model>
