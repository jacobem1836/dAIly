1. Create `src/daily/orchestrator/graph.py`:
   ```python
   from langgraph.graph import StateGraph, START, END
   from langgraph.checkpoint.memory import MemorySaver  # for testing
   from daily.orchestrator.state import SessionState

   def route_intent(state: SessionState) -> str:
       """Route based on last user message content."""
       last_msg = state.messages[-1].content.lower() if state.messages else ""
       summarise_keywords = ["summarise", "summarize", "summary", "thread", "email chain"]
       if any(kw in last_msg for kw in summarise_keywords):
           return "summarise_thread"
       return "respond"

   def build_graph(checkpointer=None):
       """Build and compile the orchestrator StateGraph.

       Args:
           checkpointer: LangGraph checkpointer instance. Use MemorySaver for tests,
                        AsyncPostgresSaver for production.
       Returns:
           CompiledStateGraph ready for ainvoke/astream.
       """
       from daily.orchestrator.nodes import respond_node, summarise_thread_node

       builder = StateGraph(SessionState)
       builder.add_node("respond", respond_node)
       builder.add_node("summarise_thread", summarise_thread_node)
       builder.add_conditional_edges(START, route_intent, {
           "respond": "respond",
           "summarise_thread": "summarise_thread",
       })
       builder.add_edge("respond", END)
       builder.add_edge("summarise_thread", END)
       return builder.compile(checkpointer=checkpointer)
   ```

   Per D-02: GPT-4.1 for summarise_thread (reasoning-heavy), GPT-4.1 mini for respond (quick follow-ups). Model selection happens inside the node functions, not in graph.py.

2. Create `src/daily/orchestrator/session.py`:
   ```python
   import asyncio
   from datetime import date
   from redis.asyncio import Redis
   from sqlalchemy.ext.asyncio import AsyncSession

   from daily.briefing.cache import get_briefing
   from daily.profile.service import load_profile
   from daily.orchestrator.graph import build_graph

   async def create_session_config(user_id: int, session_date: date | None = None) -> dict:
       """Create LangGraph config with scoped thread_id (T-03-04)."""
       d = session_date or date.today()
       return {"configurable": {"thread_id": f"user-{user_id}-{d.isoformat()}"}}

   async def initialize_session_state(
       user_id: int,
       redis: Redis,
       db_session: AsyncSession,
       session_date: date | None = None,
   ) -> dict:
       """Load cached briefing and user preferences into initial state.

       Per D-11: Reads from Redis cache, does NOT re-run pipeline.
       """
       d = session_date or date.today()
       briefing = await get_briefing(redis, user_id, d)
       preferences = await load_profile(user_id, db_session)
       return {
           "briefing_narrative": briefing.narrative if briefing else "",
           "active_user_id": user_id,
           "preferences": preferences.model_dump(),
       }

   async def run_session(graph, user_input: str, config: dict, initial_state: dict | None = None):
       """Run a single turn through the orchestrator graph.

       Uses ainvoke (not invoke) to avoid hanging with async checkpointer (Pitfall 2).
       """
       state_input = {"messages": [("human", user_input)]}
       if initial_state:
           state_input.update(initial_state)
       return await graph.ainvoke(state_input, config)
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Email content -> LLM | Raw email bodies fetched on-demand must pass through summarise_and_redact() before entering LLM context |
| LLM response -> orchestrator | LLM output must be validated as OrchestratorIntent — no arbitrary actions |
| User input -> graph routing | route_intent uses keyword matching only — no user-controlled code execution |
| Session state -> checkpointer (Postgres) | Only summaries enter state; raw bodies are local variables only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Information Disclosure | thread_id | mitigate | thread_id = f"user-{user_id}-{date}" — user_id is system-assigned, never user-supplied. Prevents cross-user state access. |
| T-03-05 | Tampering | LLM prompt injection via email | mitigate | summarise_and_redact() runs before any email content enters LLM. Redactor strips credentials and summarises — injected instructions lose context. |
| T-03-06 | Elevation of Privilege | LLM tool calls | mitigate | No `tools=` parameter on any LLM call. `response_format=json_object` constrains output. OrchestratorIntent Literal validation rejects unknown actions. |
| T-03-07 | Information Disclosure | Raw email in SessionState | mitigate | Raw body is a local variable in summarise_thread_node — never assigned to state fields. Only redacted summary enters state via AIMessage. |
| T-03-08 | Denial of Service | Large email body to redactor | accept | summarise_and_redact has a semaphore (3 concurrent calls) and max_tokens=200. Single-user Phase 3 — acceptable risk. |
</threat_model>
