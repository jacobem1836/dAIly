```markdown
# Deploying dAIly

Single-host VPS deployment using Docker Compose and Caddy.

## Prerequisites

- A VPS (Ubuntu 22.04+ recommended) with Docker and Docker Compose installed
- A domain name pointed at the VPS public IP (A record)
- API keys: OpenAI, Deepgram, Cartesia, and OAuth credentials for integrations

## 1. Clone and configure

[git clone, cp .env.example .env, edit .env with real values]
[Note: For Docker Compose, change localhost to service names in DATABASE_URL, DATABASE_URL_PSYCOPG, REDIS_URL -- the comments in .env.example show both variants]

## 2. Bind app to loopback

[IMPORTANT: Edit docker-compose.yml to change app ports from "8000:8000" to "127.0.0.1:8000:8000" so the app is only accessible via Caddy, not directly from the internet]

## 3. Start the stack

[docker compose up -d]
[docker compose logs -f app -- wait for "Starting uvicorn..."]
[docker compose ps -- verify all 3 services are healthy]

## 4. Install Caddy

[apt install commands from RESEARCH.md]

## 5. Configure Caddy

[Create /etc/caddy/Caddyfile with the 3-line reverse proxy config]
[sudo systemctl reload caddy]
[Caddy auto-obtains TLS certificate from Let's Encrypt]

## 6. Smoke test

[curl https://yourdomain.com/health]
[Expected: {"status": "ok", ...}]

## Maintenance

- View logs: docker compose logs -f app
- Restart: docker compose restart app
- Update: git pull && docker compose up --build -d
- Backup database: docker compose exec postgres pg_dump -U daily daily > backup.sql
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public Internet -> Caddy (443) | TLS termination at Caddy; untrusted traffic enters here |
| Caddy -> App (localhost:8000) | Trusted loopback; Caddy forwards to app on 127.0.0.1 only |
| VPS firewall -> Port 8000 | Must NOT be publicly accessible; loopback binding enforces this |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-09 | Spoofing | TLS termination | mitigate | DEPLOY.md mandates Caddy with auto-TLS from Let's Encrypt; no self-signed certs |
| T-15-10 | Tampering | Public port 8000 | mitigate | DEPLOY.md documents changing port binding to 127.0.0.1:8000:8000 so app is only accessible via Caddy |
| T-15-11 | Information Disclosure | .env on VPS | mitigate | DEPLOY.md instructs setting proper file permissions; .env never committed to git |
| T-15-12 | Denial of Service | No firewall guidance | accept | Firewall config (ufw/iptables) is outside scope; developer audience is expected to handle basic VPS hardening. Loopback binding mitigates the most common exposure. |
</threat_model>
